version 0.10

All

  • The model.iterations parameter no longer has effect; iteration count is no longer explicitly specified, but instead determined dynamically by a convergence criterion: the average absolute difference in estimates for user-item pairs between iterations. The model.als.iteration.convergenceThreshold parameter, which defaults to 0.001, will generally cause significantly more iterations to be run. Increase (to 0.01 for example) for fewer iterations. The number of iterations can be capped by setting model.iterations.max to the (positive) max number of iterations to run.
